Login / Protected Routes

Part 1 — REST API

Request / Response

Click the button to fetch a list of European countries from a public REST API. Your browser asks our server, which then asks the external API and sends the data back.

Part 2 — Webhooks

Event Driven

Instead of you asking for data, the external service calls your server when something happens. Click the button to simulate an event being sent — just like GitHub or Stripe would send one. Each event is signed so your server can verify it's legitimate.

Part 3 — WebSockets

Real-time / Two-way

A persistent connection where either side can send data at any time. Once connected, the server pushes live price updates every second — no requests needed. You can also send a message to the server and it will respond instantly.

Database — SQLite

Persistence

Users and webhook events are now stored in a real SQLite database file on disk. Register a user and simulate some webhook events, then restart the server — the data will still be here. The PostgreSQL version uses the same queries, just a different driver.
